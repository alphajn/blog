命令行常用代码 ​

代理 ​

设置代理 ​

bash
export ALL_PROXY=socks5://127.0.0.1:1086 // 设置所有代理
export http_proxy=http://127.0.0.1:1087 // 设置http代理
export https_proxy=http://127.0.0.1:1087 // 设置https代理

取消代理 ​

bash
unset ALL_PROXY=socks5://127.0.0.1:1086
unset http_proxy=http://127.0.0.1:1087
unset https_proxy=http://127.0.0.1:1087

设置命令行快速启动/取消 ​

  1. 在~/.zshrc文件添加以下代码

    bash
    alias proxy='export all_proxy=socks5://127.0.0.1:1086'
alias unproxy='unset all_proxy'

  2. 执行source ~/.zshrc

  3. 开启代理proxy

  4. 取消代理unproxy

ssh代理 ​

添加以下代码到~/.ssh/config文件

bash
Host github.com
ProxyCommand nc -X 5 -v -x 127.0.0.1:1086 %h %p

参数说明 ​

  • -X: 代理协议5是socks5协议
  • -v: 运行时命令行链接提示
  • -x: 指定代理服务器和端口

查看域名信息 ​

bash
curl ip.gs
# 或
curl cip.cc
# 或 添加命令行快捷方式
alias ip='curl cip.cc'

浏览器相关 ​

启动跨域浏览器 ​

  1. 桌面新建文件夹chromeData，添加以下代码到~/.zshrc文件

    bash
    alias devChrome='open -n /Applications/Google\ Chrome.app --args --disable-web-security --user-data-dir={chromeData路径}'

  2. 执行source ~/.zshrc

  3. 打开浏览器devChrome

find命令 ​

语法: find [path] [expression]

参数说明 ​

  • path: 目录或文件名，多个路径之间用空格分隔，如果未指定路径，则默认为当前目录。
  • expression: 可以是文件名、文件类型、文件大小等。
    • -name: 按文件名查找，支持使用通配符 * 和 ?。
    • -type: 按文件类型查找，可以是 f（普通文件）、d（目录）、l（符号链接）等。
    • -size [+-]size[cwbkMG]：按文件大小查找，支持使用 + 或 - 表示大于或小于指定大小，单位可以是 c（字节）、w（字数）、b（块数）、k（KB）、M（MB）或 G（GB）。

用法 ​

用法功能描述
find . -name file.txt查找当前目录下名为file.text的文件
find . -name '*.txt'查找当前目录及其子目录下所有文件后缀为txt的文件
find /home -size +1M查找 /home 目录下大于 1MB 的文件
find . -type d查找当前目录下所有的文件夹
find . -name '*.txt' | grep -r 'abc'查找当前目录下包含abc字符串的所有txt文件

which命令 ​

在PATH变量指定的路径中，搜索某个系统命令的位置，并且返回第一个搜索结果。

用法 ​

bash
which node

grep命令 ​

查找文件里符合条件的字符串或正则表达式。

用法 ​

bash
grep [options] pattern [files]

  1. 在file.txt文件中查找test字符串。

    bash
    grep test file.txt

  2. 文件夹dir中递归查找所有文件中匹配正则表达式test的行，并打印匹配行所在的文件名和行号。

    bash
    grep -rn test dir/

  3. 查找当前目录下所有包含grep字符串以.md结尾的文件。

    bash
    find . -name '*.md' | xargs grep -n 'grep'

参数 ​

  • options:
    • -i: 忽略大小写进行匹配。
    • -v: 反向查找，只打印不匹配的行。
    • -n: 显示匹配行的行号。
    • -r: 递归查找子目录中的文件。
    • -l: 只打印匹配的文件名。
    • -c: 只打印匹配的行数
    • -E: 正则表达式匹配

curl命令 ​

curl使用文档：cookbooks

用法 ​

bash
curl github.com

-G 构造 URL 的查询字符串 ​

bash
curl -G -d 'login=emma＆password=123' https://github.com/login
# 或 使用 --data-urlencode 编码 URL
curl -G --data-urlencode 'login=em ma＆password=123' https://github.com/login

-d 发送 POST 请求的数据体 ​

bash
curl -d '{"login": "emma", "pass": "123"}' -H 'Content-Type: application/json' https://google.com/login
# 或
curl -d 'login=emma＆password=123' https://github.com/login
# 或
curl -d 'login=emma＆password=123' -X POST https://github.com/login
# 或
curl -d 'login=emma' -d 'password=123' -X POST  https://github.com/login

-v 参数输出通信的整个过程，用于调试 ​

bash
curl -v github.com

-H 设置请求头 ​

bash
# 设置 User-Agent
curl -H 'User-Agent: abc/test' github.com
# 设置 Referer
curl -H 'Referer: https://google.com?a=1' github.com

-i打印服务器回应的 HTTP 标头 ​

bash
curl -i github.com
# 或 只打印相应头
curl -I github.com

-A 设置请求User-Agent ​

bash
curl -A 'abc/test' github.com

-b 发送Cookie ​

bash
curl -b 'a=1&b=2' github.com

-e 设置 HTTP 的标头Referer ​

bash
curl -e 'https://google.com?a=1' github.com

-F 上传二进制文件 ​

bash
curl -F 'file=@photo.png' https://google.com/profile

-L 参数会让 HTTP 请求跟随服务器的重定向。curl 默认不跟随重定向 ​

bash
curl -L github.com

-X 参数指定 HTTP 请求的方法 ​

bash
curl -X POST https://www.baidu.com/login
